## Plan: Mini UFlash OCR — Windows-native Gradio integration (built in-place at `D:\Python\OCR`)

### Reality check (confirmed by exploration)
- **Project root = `D:\Python\OCR`** (user confirmed build in-place; `D:\AI` is not created).
- **Python = 3.13.5** (existing `.venv`, user confirmed). Deviation from spec's "3.12" is a documented, justified choice — every dependency (torch CUDA, transformers, gradio, pymupdf) supports 3.13.
- **Unlimited-OCR model** already present: `models/PaddlePaddle/Unlimited-OCR` (6.21 GB BF16, 12-layer MoE, hidden 1280, sliding_window 128). Loaded once, shared by both modes.
- **Stage 11B weight** already present: `_mini_uflash_preview/.../drafter_v2_stage11b_best.pt` (30 MB, 7.96 M params, **pure-PyTorch SDPA** — fully Windows-native). Copy into a clean `weights/` dir (30 MB is trivial; the 6 GB model is NOT copied, just referenced).
- **Step = 41500 + 8000 = 49500** ✓ (matches spec). **Layer indices = [1,4,6,9,11]**.
- **Mini UFlash is a diagnostic layer, never the OCR output** — strict B1 replay is the commit authority (exactness guarantee). This matches the spec's "正式 OCR 文本仍采用 Unlimited-OCR 官方结果".
- **Critical enabler**: the exactness probe needs an "oracle payload" (captured token trajectory). `build_v2_teachers_from_images.py` shows how to **capture it live from any uploaded image** by wrapping `model.generate()` during one `.infer()` call. → No pre-captured files, no training machine needed. Both modes use one shared model load (feasible on 8 GB: ~6 GB model + ~30 MB drafter).

### Implementation phases

**Phase 1 — Environment & dependency install (no friction)**
- Populate `requirements-windows.txt` (gradio, pymupdf, pillow, psutil, safetensors, huggingface_hub, numpy, pytest). Pin transformers to a version compatible with this local trust-remote-code model. **No** flash-attn/triton/xformers.
- Decide the torch CUDA wheel: GPU is RTX 5060 (Blackwell sm_120) on CUDA 12.8 driver → needs a recent CUDA 12.8 build of torch. `install_windows.ps1` installs torch from the cu128 index only if torch isn't already CUDA-capable; never replaces a working CUDA torch with CPU torch (spec rules). Reuse existing 3.13 `.venv`; document the 3.13 choice in README.

**Phase 2 — Path/config/Windows-hardening layer**
- `path_utils.py`: pathlib everywhere; long-path detection (>240 warn); Chinese username/dirs/spaces/different drives; `Path.home()`.
- `process_manager.py`: PID file read/validate (by command line match), safe kill. `if __name__=="__main__":` guards on every entrypoint.

**Phase 3 — Model manager (shared single load)**
- Thread-safe singleton. `attn_implementation="sdpa"`, fallback `"eager"` (never flash-attn). **BF16** primary; detect non-BF16 GPU → clear message, no silent switch. Wrap Unlimited-OCR's `_ring_window`/`sliding_window` toggling (as the probe scripts do). Late-load (only on "加载模型"), unload with `gc.collect()` + `torch.cuda.empty_cache()`. Missing-model → exact message + directory picker.

**Phase 4 — Stable mode engine (Unlimited-OCR official path)**
- `unlimited_ocr_engine.py`: reuse the proven `.infer(... base_size, image_size, crop_mode, max_length, no_repeat_ngram_size, ngram_window, save_results ...)` API (already working in `batch_ocr.py`). Gundam/Crop = crop_mode=True, base_size=1024, image_size=640; Base = single global view. PNG/JPG/JPEG/WEBP/BMP/TIFF. Clean `<|det|>` tags.

**Phase 5 — PDF support**
- `pdf_utils.py`: PyMuPDF, `with fitz.open(...)`, per-page RGB render, page progress, single-page failure continues, Markdown page separators, temp-image cleanup, handles closed promptly.

**Phase 6 — Mini UFlash precise mode (the core research feature)**
- `mini_uflash_core/`: vendor the Stage 11B drafter (`model.py`, `checkpoint.py`) + `stage9_common.py` helpers + port `OnlineTarget` and the `run_case` B8 exactness loop from `stage9_v2_b8_exactness_probe.py`. These are already pure PyTorch — no porting of Linux deps needed.
- `payload_capture.py`: port `build_v2_teachers_from_images.py`'s `capturing_generate` trick — wrap `model.generate()` during ONE official `.infer()` call on the uploaded image to produce the live oracle payload (input_ids, output_ids, images_seq_mask, images_crop/ori/spatial_crop, generation policy). Image-only (PDF → auto-fallback to stable with a reason banner; no crash).
- `mini_uflash_engine.py`: strict B1 replay commit (block verifier is diagnostic only). Reports ALL spec metrics: checkpoint path/step, tokens, speculative rounds, mean accepted draft, effective tokens/round, full-block count/ratio, acceptance histogram, per-position accuracy (7 positions for B8), drafter/block-verifier/strict-replay latency, block-vs-B1 disagreement, final token exactness, first mismatch, theoretical decode speedup (labeled "理论估算，不是端到端实际加速"). Expected: mean accepted draft ≈ 4.27/7, full-block ≈ 33% (matches the stage11b report). On exactness failure: red warning + save first mismatch + official + strict tokens; **official text stays the OCR output**.
- Direct Block Cache: shown as "实验性 Direct Block Cache：当前禁用", no user toggle.

**Phase 7 — Output & export**
- Per-run `webapp/outputs/YYYYMMDD_HHMMSS_random/` with `input/ result.md result.txt result.json metrics.json run.log pages/`. `Path.resolve()`. JSON `ensure_ascii=False, indent=2`, schema per spec. Downloads: result.md/.txt/.json/metrics.json.

**Phase 8 — Gradio UI** (`ui.py` + `app.py`)
- Blocks, `127.0.0.1:7860`, `share=False`, reload off. Top bar + env status (Windows/Python/GPU/CUDA/engines/attn backend/mode). Left: upload, preview, model path, load, mode, Gundam/Base, max tokens, probe token count, run, clear, unload. Right tabs: Markdown / 纯文本 / 原始输出 / Mini UFlash 指标 / 运行日志. Fog-white/light-grey, large radius, restrained shadows, subtle glass, dark mode, no cheap gradients. Page opens before any model load.

**Phase 9 — PowerShell scripts**
- `install_windows.ps1` (ExecPolicy Bypass in-process, `Join-Path $PSScriptRoot .venv\Scripts\python.exe`, `$ErrorActionPreference="Stop"`, exit codes). `launch_webapp.ps1` (`-NoBrowser`, PID file, dup-start guard, port check, `& $Python -m webapp.app`). `stop_webapp.ps1` (PID-file only, validate cmdline belongs to this project, never `taskkill /IM python.exe /F`). `check_environment.ps1`, `download_models.ps1`.

**Phase 10 — Tests & verification**
- Static: `compileall webapp`; parse-check all `.ps1`. Unit (`webapp/tests`): Windows drive/Chinese/space paths, model-dir detection, checkpoint auto-find (priority Stage11B>11A>30k; never `_last`), output-dir creation, Markdown clean, JSON export, PDF page convert, handle release, metrics format, exactness mismatch, PID file mgmt. Startup test: `launch_webapp.ps1 -NoBrowser` → `Invoke-WebRequest http://127.0.0.1:7860` = HTTP 200. Real inference test if model loads: stable OCR on a CN/EN test image, then Mini UFlash precise mode, check step≈49500 + exactness + record VRAM/time. If model missing: page still starts, path picker works, missing-path message accurate, no faked OCR.

**Phase 11 — GitHub readiness**
- `.gitignore` excludes `.venv/ models/ weights/*.pt *.safetensors webapp/outputs/ logs/ __pycache__/ webapp.pid`. README documents: Windows-native only, NVIDIA GPU required, Unlimited-OCR is stable engine, Mini UFlash strict replay is precise research mode, Direct Block Cache disabled, ~2.4× is theoretical decode potential not end-to-end speed. Stage 11B weight → GitHub Release.

### Hard constraints honored
Windows-native only (no WSL/Docker/Bash-launch); SDPA→eager (no flash-attn/triton dependency); pure-PyTorch for the drafter; pathlib + `with` everywhere; `__main__` guards; real model inference (no static/fake data); strict B1 replay commit (exactness); official text is the OCR truth; Direct Block Cache disabled.
